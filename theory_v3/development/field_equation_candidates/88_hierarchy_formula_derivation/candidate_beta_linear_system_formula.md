# candidate_beta_linear_system_formula — Analysis Note

## Result

`candidate_beta_linear_system_formula.py` converts the moment-ratio conditions into a Beta-function linear system.

For:

```text
P_N(t) = 1 + Σ a_j t^j
```

the moments are:

```text
I_k(P_N) = B(k+1/2, 5) + Σ a_j B(k+j+1/2, 5)
```

The constraints become:

```text
A_N a = b_N
```

For `N=4`, the script computes the exact matrix/vector and obtains:

```text
a = [-4332/131, 51186/131, -166060/131, 163875/131]
```

so:

```text
P_4(t) =
1 - (4332/131)t
+ (51186/131)t^2
- (166060/131)t^3
+ (163875/131)t^4.
```

## Interpretation

This is the main formula-level advance of Group 88.

The hierarchy now has a finite-N coefficient generator:

```text
build A_N and b_N from Beta moments;
solve A_N a = b_N.
```

That is much stronger than direct symbolic integration for each `N`. It is also the reason the patched scripts run faster: the Beta values can be evaluated as exact rational numbers instead of leaving SymPy to chew symbolic special functions.

The important conceptual result is:

```text
the finite hierarchy is a Beta-moment linear algebra problem.
```

## What Changed

The profiles are no longer separately discovered. They are generated by one rule.

This should update the status from:

```text
finite hierarchy supported by examples
```

to:

```text
finite hierarchy generated by Beta linear system.
```

## What Did Not Change

The system is conditional on invertibility:

```text
det(A_N) != 0.
```

The script explicitly preserves that all-`N` invertibility is a separate obligation.

## Steering Consequence

The next required result is a determinant/Cramer formula. That tells us exactly what needs to be proven for all-order existence and uniqueness.
