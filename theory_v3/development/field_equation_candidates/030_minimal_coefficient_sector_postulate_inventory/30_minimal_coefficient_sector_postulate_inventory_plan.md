## Candidate Minimal Postulate Families

### P1: Trace-Normalization Postulate

Possible content:

```text

```

Purpose:

```text

```

```text

```

### P2: Safe-Trace Membership Postulate

Possible content:

```text

```

Purpose:

```text
upgrade constrained candidate to explicit membership rule.
```

Risk:

```text
mistaken for no-overlap or residual control.
```

Possible content:

```text
I(T_zeta,R_zeta)=0 and/or I(T_zeta,R_kappa)=0.
```

Purpose:

```text
supply missing no-overlap relation.
```

Risk:

```text
too strong;
hides residuals;
assumes the theorem we wanted.
```

### P4: Source No-Double-Counting Postulate

Possible content:

```text
ordinary source load enters only once and is not duplicated through coefficient/accounting sectors.
```

Purpose:

```text
protect count-once recombination.
```

Risk:

```text
source routing chosen by repair.
```

### P5: Guardrail Visibility Postulate

Possible content:

```text
boundary/current/mass/support loads remain visible and cannot be absorbed into coefficient, accounting, or correction terms.
```

Purpose:

```text
protect field-equation honesty.
```

Risk:

```text
visibility mistaken for neutrality.
```

### P6: Divergence Explicitness Postulate

Possible content:

```text
any divergence correction must be explicit, auditable, and non-reservoir.
```

Purpose:

```text
avoid hidden source/boundary/current/mass/support load.
```

Risk:

```text
correction becomes parent-fit closure.
```

## Success Criteria

A postulate candidate survives only if it is:

```text
explicitly marked as postulate or choice;
minimal;
non-recovery-selected;
non-repair-selected;
not residual cleanup;
not active-O convenience;
not parent-fit closure;
compatible with volume/trace partial coefficient origin;
compatible with zeta_Bs -> T_zeta constrained candidate status;
compatible with residual visibility;
compatible with source visibility;
compatible with guardrail visibility;
compatible with explicit divergence obligations;
not enough by itself to claim insertion unless all insertion gates are separately satisfied.
```

---

## Failure Criteria

A candidate fails if it:

```text
smuggles recovery;
hides residuals;
duplicates source;
absorbs boundary/current/mass/support load;
uses divergence correction as reservoir;
asserts no-overlap by naming;
asserts insertion by naming;
opens active O;
opens residual control;
opens parent equation.
```
