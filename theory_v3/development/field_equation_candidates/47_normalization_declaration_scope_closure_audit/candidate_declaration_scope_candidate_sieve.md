# Candidate Declaration Scope Candidate Sieve — Result

## Result

The sieve identifies the strongest surviving scope route: a limited paired-record declaration-scope candidate. This candidate is restricted to the paired non-active metric and scale record surface.

It rejects or defers broader routes. Neutral-law declaration scope is eliminated. Single-branch scope is choice-required. Parent-facing scope is theorem-required. Insertion-facing scope is not ready.

## Main Findings

This is the main branch-killing result of the Group 47 batch. The surviving declaration-scope route is not a branch choice and not a parent-facing route. It is a limited paired-record scope that preserves both `B_s_metric` and `b_s_scale` as branch-indexed candidates.

The neutral route is dead: the paired records cannot be collapsed into unqualified `B_s` or neutral `F_zeta` with an expression. That keeps the factor-of-two burden alive.

The single-branch route is not dead, but it requires a separate explicit choice. That means it cannot be used as the Group 47 outcome.

The parent and insertion routes remain unavailable because they require theorem support and scalar-response licensing outside this scope audit.

## Boundary

The surviving scope candidate is not declaration itself. It is ready for a future scope/status record. It does not install `log(B_s_metric)=2*zeta/d`, does not install `log(b_s_scale)=zeta/d`, and does not make either branch active.

## Safe Handoff

Carry forward the limited paired-record declaration-scope candidate. The next artifact should state its status, assumptions, domain, numeric `d` condition, non-active branch status, and downstream caveats.
