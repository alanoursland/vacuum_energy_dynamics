## Planning note

This group is still close enough to the current state to plan concretely, but it should be treated as an exclusion / mapping group, not a decision group. Its job is to reduce the space of possible equations and future axioms by making bad choices visible, not to choose the final trace-anchor equation.

The farther roadmap remains speculative. This group should be written so that it can stand on its own even if later group numbers change.

---

## Why this is the next group

Group 41 sharpened the safe-membership side of the trace-anchor surface:

```text
zeta_Bs object slot
T_zeta safe trace-sector target
domain/codomain and criterion needs
role-purity exclusion zones
diagnostic-versus-active boundary
```

But Group 41 did not install active membership, did not complete a safe-membership declaration, did not choose a B_s branch, did not prove incidence or residual control, did not adopt Package B, did not open active O, did not insert B_s/F_zeta, and did not open parent closure.

The wider field-equation snapshot says the strongest current result remains the reduced static spherical A-sector areal-flux law, while the central unresolved blocker is licensed recombination:

```text
A-sector mass response
B_s/F_zeta scalar spatial response
zeta/kappa residual status
source routing
boundary behavior
divergence safety
parent identity
```

The system remains underconstrained. Therefore the next honest move is not to choose an equation, but to map and eliminate candidate equation families that are already disallowed by the current guardrails.

## Equation-family buckets to classify

### 1. Trace-normalization equation families

Examples to classify, not choose:

```text
metric-coefficient branch:
  log(B_s_metric) = 2 zeta / d

scale-factor branch:
  log(b_s_scale) = zeta / d

parallel branch-indexed declarations:
  keep both as explicit non-active candidate forms

invalid neutral form:
  write one unqualified B_s or neutral F_zeta expression that hides the factor-of-two burden
```

Main questions:

```text
Does the equation depend on branch choice?
Does it hide branch choice under neutral notation?
Does it derive from recovery rather than declaration/axiom?
Does it remain only branch-indexed candidate form?
```

### 2. Safe-membership equation / relation families

Examples to classify:

```text
zeta_Bs -> T_zeta as diagnostic compatibility label
zeta_Bs_metric -> T_zeta_metric as branch-indexed candidate
zeta_Bs_scale -> T_zeta_scale as branch-indexed candidate
membership as active O or projector
membership as residual nonentry
membership as source no-double-counting
membership as insertion support
```

Main questions:

```text
Is this only diagnostic / compatible-if-declared?
Does it require a later declaration?
Does it pretend to prove incidence or residual control?
Does it smuggle active O?
Does it require a new membership axiom?
```

### 3. B_s/F_zeta scalar spatial-response equation families

Examples to classify:

```text
equations selected from AB=1 or B=1/A
equations selected from Schwarzschild recovery or PPN gamma
equations using neutral F_zeta while installing a concrete expression
equations with branch-indexed B_s/F_zeta candidate form but no insertion
equations that need an explicit new insertion axiom
```

Main questions:

```text
Is the equation chosen from recovery?
Does it duplicate A-sector mass response?
Does it create exterior scalar tail?
Does it carry source or boundary payload?
Is it insertable yet?
```

### 4. Residual-control equation families

Examples to classify:

```text
residual killed structurally
residual strictly inert / non-metric
residual excluded by membership
residual removed by active O
residual ignored
```

Main questions:

```text
Is residual nonentry actually proved?
Is residual killed by declaration?
Is active O being used as an eraser?
Does residual re-enter metric trace?
```

### 5. Source-routing and mass-protection equation families

Examples to classify:

```text
ordinary mass response only through protected A-sector coin

```

Main questions:

```text
Does the equation duplicate M_A?
Does it hide ordinary source load?
Does it preserve A-sector mass neutrality outside the source?
Does it require a new source-routing axiom?
```

### 6. Boundary / support / scalar-tail equation families

Examples to classify:

```text
compact-support scalar response
boundary-neutral scalar response
shell-source repair
far-zone scalar-tail suppression by filter
boundary counterterm
```

Main questions:

```text
Is boundary neutrality derived before recovery?
Is a shell source being introduced without source law?
Is scalar silence imposed as a patch?

```

### 7. Divergence / parent-identity equation families

Examples to classify:

```text

```

Main questions:

```text

Does the equation open parent closure early?
Is parent-readiness being inferred from reduced recovery?
```
