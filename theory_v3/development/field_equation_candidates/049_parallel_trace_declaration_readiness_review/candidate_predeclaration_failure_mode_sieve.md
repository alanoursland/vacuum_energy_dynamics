# Candidate Predeclaration Failure Mode Sieve — Result

## Result

This script attacks the possible declaration attempt before it is written. The result is useful: many broadened declaration routes are killed, but the limited symbolic paired caveated attempt survives.

## Main Findings

The failure modes are clear. A future declaration attempt fails or is demoted if it smuggles a branch choice, collapses the pair into a neutral law, leaks a numeric `d`, uses recovery as support, or drifts into insertion, active `O`, safety theorem, recombination, or parent use.

The surviving shape is narrow: a symbolic paired declaration attempt with explicit caveats. It may be considered next, but surviving this sieve is only permission to attempt, not proof or declaration.

This is exactly the kind of branch-killing script that helps the field-equation path. It eliminates false shortcuts while preserving the one disciplined route that remains available.

## Boundary

The sieve does not declare trace normalization. It does not prove the paired route true. It does not make the later attempt succeed automatically.

## Steering Conclusion

The future declaration attempt should be adversarially checked against this failure list. Any attempt that drops branch labels, fixes numeric `d`, uses recovery, or opens downstream machinery should be rejected.
