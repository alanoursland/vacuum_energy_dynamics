# Candidate Branch Decision Obligation Matrix

## Result

This script mapped the obligations attached to the metric-choice, scale-choice, parallel-record, and continued-deferral routes.

It did not close those obligations and did not select a route.

## Main Findings

The metric and scale routes each require object/scope clarity, `zeta` convention, traced dimension, a valid selector record, and explicit downstream caveats.

The parallel-record route requires branch labels to remain explicit, expressions to stay separated, and no collapse into neutral notation or unqualified `B_s`.

Continued deferral remains a valid route if branch support is insufficient or if theorem/axiom work should run first. Deferral preserves options; it is not a branch no-go theorem.

The matrix also keeps trace-normalization declaration, Package B adoption, and `B_s/F_zeta` insertion as separate downstream obligations.

## Boundary

Obligation visibility is not obligation closure.

The script rejects choosing the branch with fewer listed burdens, treating deferral as branch failure, completing trace normalization from an obligation matrix, or adopting Package B because route burdens are now visible.

## Safe Handoff

Run `candidate_branch_or_parallel_decision_batch_reconciliation.py` next.

The reconciliation should check whether the batch preserved route separation, selector discipline, open obligations, and downstream locks.
