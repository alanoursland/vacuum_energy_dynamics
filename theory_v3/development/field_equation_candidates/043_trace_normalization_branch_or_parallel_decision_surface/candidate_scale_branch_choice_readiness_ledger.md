# Candidate Scale Branch Choice Readiness Ledger

## Result

This script inventoried what would be required before `b_s_scale` could be chosen as the trace-normalization branch.

It did not choose `b_s_scale`. It kept the scale branch as a future explicit-choice candidate only.

## Main Findings

The scale candidate form

```text
log(b_s_scale)=zeta/d
```

may remain visible for a later explicit choice record.

Before that choice could support a declaration, the scale-factor object and scope must be defined, the `zeta` convention must be explicit, the traced dimension `d` must be named, and the normalization scope must be clear.

The script allows determinant/root or scale-factor intuition to be recorded as context. That context may support a later explicit theory-owner choice, but it is not a derivation and does not choose the branch by itself.

## Boundary

The scale branch cannot be chosen from determinant slogans, recovery targets, PPN gamma, `AB=1`, `B=1/A`, Schwarzschild matching, or parent-fit convenience.

The candidate expression is not a declaration. A scale branch choice, even if later made, would still not license `b_s_scale/F_zeta` insertion, residual control, or parent closure.
