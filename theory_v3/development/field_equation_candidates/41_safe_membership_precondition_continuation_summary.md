### Current membership remains compatible-if-declared only

The relation

```text
zeta_Bs -> T_zeta
```

may currently be used only as a diagnostic compatibility label or a compatible-if-declared surface.

It is not:

```text
declaration-ready
selected
adopted
derived
incidence
residual control
active O
B_s/F_zeta insertion
parent readiness
```

A later route may evaluate whether the sharpened preconditions support a declaration or theorem attempt, but Group 41 does not start that route.

### `zeta_Bs` is the membership-test object

Group 41 sharpened `zeta_Bs` as the payload object being tested for membership.

It is not:

```text
F_zeta
B_s/F_zeta insertion
trace-normalization expression
branch-choice mechanism
residual/source/correction payload
```

Where branch sensitivity is absent, `zeta_Bs` may remain a shorthand object label. Where branch sensitivity matters, the object must be split explicitly:

```text
zeta_Bs_metric
zeta_Bs_scale
```

These are parallel non-active membership-object candidates. They do not choose `B_s_metric` or `b_s_scale` and do not complete trace normalization.

### `T_zeta` is a safe trace-sector target

Group 41 sharpened `T_zeta` as a safe trace-sector target or diagnostic sector label for possible later membership testing.

It is not:

```text

projector

incidence theorem
no-overlap theorem

parent-equation object
```

The current safest status is diagnostic. Conditional variants may be carried only if explicitly labeled:


The future-active label is only a route label, not a current sector declaration.

### Membership-testable means precondition clarity only

Group 41 defined “membership testable” conservatively.

It means:

```text
the preconditions are explicit enough to know what a later theorem or declaration would need
```

It does not mean:

```text
theorem-ready
declaration-ready
adopted
derived
insertable
parent-ready
```

The criterion matrix separated:

```text
type criterion
trace-payload criterion
role-purity criterion
branch-index consistency criterion
diagnostic-inertness criterion
exclusion-zone criterion
declaration-support criterion
theorem-support criterion
```

These criteria are preconditions. They are not proofs.

### Role purity is mandatory but not proved

Group 41 stated the mandatory exclusion zones for any later safe-membership use.

Membership must not carry:

```text
residual payload
ordinary source payload
A-sector mass payload
correction tensor payload
boundary or shell payload
divergence repair payload
trace-normalization / branch-choice payload
recombination payload
active O or projector payload
B_s/F_zeta insertion payload
parent closure payload
```

These exclusions are policy/precondition boundaries. They do not prove no-overlap, source no-double-counting, residual nonentry, divergence safety, boundary neutrality, or parent closure.

### Diagnostic membership remains inert

The safest current status is:

```text
diagnostic_label
```

A diagnostic safe-membership label may record object/sector compatibility only.

It has no:

```text
source effect
metric effect
divergence effect
projection effect
insertion effect
theorem effect
adoption effect
parent-equation effect
```

Group 41 separated the status ladder:

```text
diagnostic_label
compatible_if_declared
candidate_if_preconditions_met
future_declaration_candidate
future_theorem_target
active_membership_forbidden_here
```

The future labels remain future. Active membership is forbidden in this group.
