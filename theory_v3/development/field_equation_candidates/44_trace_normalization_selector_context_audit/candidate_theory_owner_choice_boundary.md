# Candidate Theory-Owner Choice Boundary

## Result

This script defined what a later explicit theory-owner convention choice would need to say if intuition is used to choose a trace-normalization route.

The important result is that intuition is allowed only in daylight. A future branch or parallel-route decision may be made as a convention, but it must be explicitly labeled as a choice and must not pretend to be a derivation.

## Main Findings

- A future choice record must name the chosen route: metric, scale, parallel, or deferral.
- If the choice is not derived, the record must say it is a choice or convention.
- A future choice must disclaim forbidden selectors: recovery, downstream convenience, neutral-expression selection, and majority-count selection.
- A future choice must list remaining object/scope, `zeta`, dimension, source, residual, boundary, and insertion obligations.
- A branch choice alone would not open insertion, active `O`, recombination, or parent closure.
- A future choice record should state what later theorem failure would force reconsideration or deferral.

## Boundary

This script defines the shape of a future explicit choice record. It does not make that choice.

A convention choice is not a theorem, not Package B adoption, not trace-normalization declaration by itself, and not `B_s/F_zeta` insertion support.

## Steering Note

This is the clearest bridge from audit work into actual theory choice. It makes room for intuitive judgment without letting intuition masquerade as mathematics. If the project decides to choose a branch soon, this script is the checklist that keeps the choice honest.
