# Candidate Source Hierarchy Evidence Ledger

## Result

This script separated usable source-context from weak or forbidden notation evidence.

The main result is that source and notation evidence may inform a later trace-normalization route only after it is ranked by authority, recency/currentness, and relevance to the repaired notation state. Majority hit count and old overloaded `B_s` usage are not valid selectors.

## Main Findings

- Earliest explicit internal definitions can be high-value context because they may reveal intended object meaning.
- The current snapshot and governance records may outrank old shorthand because they reflect the repaired `B_s_metric` / `b_s_scale` split.
- Script outputs and archive markers are useful for current-status context, but they are not theorem proofs.
- Result summaries are navigation artifacts; they can summarize accepted state but cannot act as proof or branch selectors.
- Old unqualified `B_s` usage is weak context because it predates notation repair and may hide the original overload.
- Majority notation count is rejected as a selector because it ignores authority, age, and overload risk.

## Boundary

Evidence may inform a later explicit choice, but it cannot make the choice. The repaired split notation must not be collapsed back into old unqualified `B_s` because an older text or majority of references used that shorthand.

The script does not rank a complete source hierarchy yet. It only defines how such evidence should be treated if used later.

## Steering Note

This result weakens any future branch choice based on "what the notation usually says." The only evidence route that remains credible is a quality-ranked source hierarchy. That means a later explicit branch-choice record should either include that hierarchy or admit that the choice is a convention rather than an evidence-derived result.
