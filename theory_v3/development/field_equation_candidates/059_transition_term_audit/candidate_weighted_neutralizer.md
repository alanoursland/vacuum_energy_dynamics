# candidate_weighted_neutralizer — Result Note

## Result

The script derives a reduced weighted-neutralization operator for candidate scalar layer profiles.

The operator is:

```text
N_w[f] = w(y)*(f(y)-mu_w[f])
```

with:

```text
mu_w[f] =
integral r^2*w*f dy
/
integral r^2*w dy
```

For the important test case:

```text
f(y)=y
```

the script obtains:

```text
mu_w[y] = 2Rell/(7R^2+ell^2)
```

and therefore:

```text
N_w[y] = (y^2-1)^2*(-2Rell + y*(7R^2+ell^2))/(7R^2+ell^2)
```

with:

```text
integral r^2*N_w[y] dy = 0
```

It also shows the constant case degenerates:

```text
N_w[1]=0
```

## Main Findings

This is the strongest constructive result in Group 59.

Group 58 gave one weighted-neutral profile. Group 59 generalizes it into a reduced neutralization operator.

That means weighted-neutral candidate profiles do not have to be hand-built one at a time. They can be generated by subtracting the weighted mean with respect to the layer window and spherical measure.

The operator also reproduces the exact Group 58 skew:

```text
c*=2Rell/(7R^2+ell^2)
```

So the new construction is consistent with the previous group, not a separate trick.

## Boundary

`N_w` is not active `O`. It is not a covariant projection. It is not an insertion law. It only neutralizes reduced scalar layer profiles under the weighted spherical measure.

## Steering Consequence

The next filter must test whether weighted-neutral candidates are source-clean and trace-clean. Weighted charge neutrality does not prove source safety.
