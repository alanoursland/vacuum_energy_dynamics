# candidate_uniqueness_and_scaling_test — Analysis Note

## Result

`candidate_uniqueness_and_scaling_test.py` tests uniqueness and scaling.

The weighted charge is:

```text
-1024 ell(2R c - 3ell)/3465
```

Its derivative with respect to `c` is:

```text
-2048 R ell / 3465
```

So for nonzero `R` and `ell`, the condition is genuinely linear in `c` and has a unique solution:

```text
c = 3ell/(2R)
```

Under:

```text
ell = eps R
```

the skew becomes:

```text
c = 3eps/2
```

and in the thin-layer limit:

```text
eps -> 0
c -> 0
```

## Interpretation

This strengthens the geometric reading.

The skew is not an arbitrary dimensionless constant. It scales with the layer-to-radius ratio:

```text
c ~ ell/R
```

That is exactly what one would expect for a finite-thickness geometric correction. The skew vanishes in the thin-layer limit, meaning the symmetric compact-support exactness model is recovered when the measure becomes effectively flat.

This is a good sign. It suggests the skew is not an unnatural patch but a controlled correction generated by nonconstant measure geometry.

## What Changed

The skew is now:

```text
unique inside the linear-skew family;
dimensionless;
thin-limit consistent;
geometrically scaled.
```

That reduces the repair concern further.

## What Did Not Change

The uniqueness is not universal. It is unique only within the chosen linear-skew family:

```text
Xi = f(1 + cy)
```

A different shape family may have different moment ratios or require more parameters. This script does not prove robustness across all admissible compact-support profiles.

## Steering Consequence

A future robustness group could test whether different choices of `f,w` produce the same leading ratio or a related geometric scaling. If the coefficient is shape-dependent, the route may need an additional shape-selection principle.
