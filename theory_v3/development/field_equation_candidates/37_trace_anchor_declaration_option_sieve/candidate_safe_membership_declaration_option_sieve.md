# Candidate Safe-Membership Declaration Option Sieve

## What this run is

This run classifies safe-membership declaration options for Group 37.

It asks which ways of declaring `zeta_Bs -> T_zeta` are coherent enough to carry into a joint declaration package sieve. It does not install a declaration value as theory state, select a membership form, adopt Package B, derive a theorem, or open insertion.

## Main result

The run separates declaration-ready safe-membership options from compatibility-only, diagnostic-only, and rejected hidden-load options.

```text
typed trace-sector membership: declaration-ready option
role-pure trace-payload membership: declaration-ready option
normalization-compatible membership: candidate option only
diagnostic-only membership label: inert-only
residual-inclusive membership: rejected shortcut
source/correction-inclusive membership: rejected shortcut
```

## Declaration-ready options

Typed trace-sector membership is coherent if `zeta_Bs`, `T_zeta`, domain, codomain, and membership criterion are all explicit.

Role-pure trace-payload membership is coherent if residual, ordinary-source, correction/divergence, boundary/support, and downstream payloads are explicitly excluded.

Both are options only. Neither is selected, adopted, or derived.

## Fenced options

Normalization-compatible membership may survive only as a coherence option. Normalization cannot choose membership, and membership cannot choose normalization.

Diagnostic-only membership labels are safe only if strictly inert. They cannot alter equations, project sectors, support insertion, or become active membership by prose drift.

## Rejected options

Residual-inclusive membership is rejected because it hides residual control.

Source/correction-inclusive membership is rejected because it turns membership into a reservoir for ordinary source, correction, divergence, boundary, support, or downstream load.

## Rejected upgrades

The run rejects:

```text
membership as incidence
membership as residual control
membership as source theorem
diagnostic as active
```

## Final status

```text
Safe-membership declaration options are classified.
Typed and role-pure options are declaration-ready options.
Diagnostic labels remain inert-only.
No membership form is selected, adopted, or derived.
Current status remains compatible-if-declared only.
Downstream gates remain closed.
```
