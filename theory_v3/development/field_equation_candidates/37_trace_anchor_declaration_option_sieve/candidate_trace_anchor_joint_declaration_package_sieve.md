# Candidate Trace Anchor Joint Declaration Package Sieve

## What this run is

This run combines trace-normalization and safe-membership declaration options into joint Package B declaration-package options.

It classifies packages that could be presented to a later explicit declaration record. It does not choose a package as the theory declaration, assign component status, adopt Package B, derive either component, or open insertion.

## Main result

The run finds several coherent declaration-ready package options:

```text
scale-factor plus typed-membership package
metric-coefficient plus typed-membership package
scale-factor plus role-pure package
metric-coefficient plus role-pure package
```

It also separates diagnostic-only, mixed-scope, incomplete, and rejected hidden-load packages.

## Declaration-ready packages

The scale-factor plus typed-membership package is coherent if scale-factor `B_s` convention and typed membership criterion are explicit.

The metric-coefficient plus typed-membership package is coherent if metric-coefficient `B_s` convention, the factor of two, total trace, dimension, and membership criterion are explicit.

The scale-factor plus role-pure package is a stronger safety-fenced option if exclusion zones and diagnostic/active scope are declared.

The metric-coefficient plus role-pure package is also a stronger safety-fenced option if factor-of-two and role-purity fences are explicit.

All four are declaration options only. None is selected, adopted, recommended, or derived.

## Other package classes

The diagnostic-only package is safe for audit only. It cannot become active Package B use.

The mixed-scope package is deferred or conditional. It must carry visible mixed status and must not be reported as uniformly declaration-ready.

The incomplete package is not ready for explicit declaration if any required declaration slot remains blank.

The hidden-load package is rejected as a current declaration package because Package B components cannot hide residual, source, divergence, insertion, or parent payload.

## Rejected upgrades

The run rejects:

```text
one-node licensing
diagnostic package as active
incomplete package as declaration-ready
declaration-ready as adopted
```
