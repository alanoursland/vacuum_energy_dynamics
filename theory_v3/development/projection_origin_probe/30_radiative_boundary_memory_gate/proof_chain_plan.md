# Radiative Boundary Memory Gate Proof Chain Plan

## Purpose

This folder continues after `weyl_tensor_origin_gate` and `boundary_symplectic_closure`.
Those folders established that scalar boundary charge does not see the Weyl/TT sector,
and that the dynamical boundary ledger is symplectic rather than purely Coulombic.

This folder tests the next gate:

```text
How do radiative TT boundary data, news, memory, and flux fit into the boundary ledger
without being confused with scalar monopole charge?
```

## Strategy

The folder deliberately keeps the result conditional and local to the weak/radiative
boundary sector. It does not derive full nonlinear radiation theory. It proves the
algebraic and ledger distinctions needed for the larger framework:

```text
TT shear/news data -> radiative flux and memory;
scalar monopole charge -> Coulombic constraint ledger;
these are distinct ledgers.
```

## Batches

### Batch A: News, memory, and TT data

Show that trace-free shear/news tensors can be nonzero while scalar trace vanishes.
Show that memory is the time integral of news and can be nonzero even when initial
and final news vanish.

### Batch B: Radiative flux versus scalar charge

Show that radiative flux is quadratic in news and positive for nonzero news, while
scalar Coulomb charge can remain unchanged. This prevents double-counting radiation
as a scalar monopole source.

### Batch C: Polarization and component structure

Show plus/cross polarizations, tensor memory components, and scalar blindness to
traceless tensor shifts.

### Batch D: Ledger conclusion

Close the conditional result: radiative memory is tensor boundary displacement caused
by integrated news. It is not a scalar `r_k` charge and cannot be recovered from the
monopole boundary ledger.

## Expected conclusion

```text
Scalar boundary charge is a constraint ledger.
Radiative news/memory is a tensor symplectic boundary ledger.
```

The folder strengthens the separation between Newtonian scalar boundary admissibility
and the full GR radiative/Weyl sector.
