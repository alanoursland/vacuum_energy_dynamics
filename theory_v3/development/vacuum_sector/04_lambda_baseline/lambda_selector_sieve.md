# Lambda Selector Sieve

This sieve checks whether any chartered `Lambda` selector is ready to open a
specific nonzero-baseline mechanism.

It does not reject the selector classes. It only blocks mechanism opening until
there is actual selector evidence.

## Required Evidence

Each selector must supply:

```text
selector object;
boundary instantiation;
sign/value derivation;
source conservation;
local-equation quarantine;
operational falsifier.
```

## Current Sieve Result

| selector | first missing object | disposition |
| --- | --- | --- |
| variational minimum | explicit variational object and varied variable | not mechanism-ready |
| admissibility or boundary selector | specific boundary class replacing asymptotic flatness | not mechanism-ready |
| topology or global constraint | global constraint equation with variational or admissibility role | not mechanism-ready |
| measure identity | covariant measure identity with a conserved source ledger | not mechanism-ready |
| relaxation or nonlocal history | kernel, domain, and fixed-point equation | not mechanism-ready |
| frustration-floor microphysics | microstate variable and coarse-graining map | not mechanism-ready |

The current binary score is adequate while every row lacks an instantiated
selector object. Once a future row has partial evidence, the sieve should move
to an absent/partial/complete ledger.

## Working Conclusion

No selector row currently opens a nonzero-`Lambda` mechanism. The first
concrete handoff is the variational-minimum probe because it needs the least
additional structure before a mathematical check can be attempted.

## Non-Conclusions

This sieve does not:

```text
prove Lambda = 0 globally;
derive nonzero Lambda;
kill all boundary, topology, measure, relaxation, or microphysical routes;
license a dark-sector excess;
modify the closed local gravitational equations.
```
