# Predictions Register

The theory's observable commitments, in one place, with their kill
directions and current experimental status.

```text
predictions_register.md    the register itself, tiered by sharpness
```

House rules for this directory: an entry must state (i) what the theory
commits to, (ii) which postulate/derivation forces it, (iii) what
observation would kill it, (iv) current experimental status, and (v)
what open obligation, if any, could change the entry. No entry may be
softened after an adverse result except by an explicitly recorded
postulate-level move (the P7'-appeal pattern). Backsolving is forbidden
here as everywhere.
