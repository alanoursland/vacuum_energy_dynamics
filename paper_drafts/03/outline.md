# Paper 3 Outline: Author-Precision Data Recovery from Vector Figures

## Working Title

**The Data Is Already in the PDF: Recovering Numerical Results from
Vector Graphics in Published Figures**

Shorter title option:

**Recovering Numerical Data from Vector PDF Figures**

## Central Claim

Many published physics figures are vector graphics. Their plotted curves
are stored in the PDF as drawing paths generated by the authors' plotting
software. By extracting those paths, calibrating axes from vector tick
geometry, and validating against text-stated anchor values, one can
recover numerical data at plotting precision, often more accurately than
pixel digitization.

## Intended Reader

Experimentalists, phenomenologists, data curators, and theorists who need
numbers from published plots when tables are unavailable.

The paper should be practical, reproducible, and conservative about data
ethics.

## Abstract Skeleton

Published plots often contain more numerical information than their
captions or associated tables expose. When figures are embedded as vector
graphics, curves and filled regions are represented by PDF path
coordinates rather than pixels. We describe a reproducible method for
recovering such data: extract drawing paths from the PDF, identify the
curve-bearing path, calibrate the plot frame using tick geometry, transform
path coordinates into data coordinates, and validate the result against a
text-stated anchor value. In a case study on the Lee et al. 2020 Yukawa
exclusion figure, the method recovers the published alpha = 1 crossing at
38.6 micrometers to within 0.03% and reads an unpublished alpha = 1/3
crossing at 54.05 micrometers from the same author-generated path. A later
supplemental-table extraction gives 54.03 micrometers, providing an
independent cross-check. We
compare the method to pixel digitization, discuss failure modes, and
propose a validation-or-refuse norm for reuse of data extracted from
published vector figures.

### 3. Method

Pipeline:

1. Extract page drawings with `PyMuPDF.get_drawings()` or equivalent.
2. Identify candidate curve paths by geometry, color, stroke/fill type,
   z-order, and bounding box.
3. Identify plot frame and tick marks.
4. Calibrate PDF coordinates to data coordinates.
5. Transform curve vertices.
6. Validate against an external anchor value.
7. Emit data only if validation passes.

Important design rule:

The tool should refuse to emit "physics-grade" data if the anchor check
fails.

### 4. Axis Calibration

Goal:

Make the calibration method explicit enough to reproduce.

Content:

- Linear axes: affine mapping from PDF coordinate to data coordinate.
- Log axes: tick positions evenly spaced by log decade in PDF space.
- Calibration from tick geometry can be independent of OCR labels when
  decade structure is identifiable.
- Handling inverted y-axis / PDF coordinate conventions.
- Error propagation from tick location uncertainty to data coordinates.

Equations:

- `x_data = a x_pdf + b` for linear axes.
- `log10 x_data = a x_pdf + b` for log axes.

### 9. Provenance, Ethics, and Citation

Key points:

- Extracted data remains data from the original publication; cite the
  source paper.
- Author-requested data is the gold standard when available.
- Vector extraction is an improved form of digitization, not a replacement
  for data sharing.
- Do not overstate precision beyond plotting precision.
- Retain extraction logs, script version, source PDF, page, path ID,
  calibration, and validation anchor.

### 10. Limitations and Failure Modes

Content:

- raster-only PDFs;
- curves converted to outlines or heavily post-processed;
- missing or ambiguous ticks;
- multiple overplotted paths;
- clipping masks;
- smoothed curves where vertices are control points rather than sampled
  data;
- semantic ambiguity: plotted visual boundary may not equal raw analysis
  data.

### 11. Conclusion

Takeaway:

For many modern physics figures, the numerical data are already present
inside the PDF. A path-level extraction plus calibration and validation
workflow can recover them more precisely and more reproducibly than pixel
digitization, while preserving provenance discipline.

## Claims To Avoid

- Do not claim recovered points are the authors' original raw data.
- Do not claim path extraction always beats author-provided data.
- Do not ignore legal/ethical provenance.
- Do not emit unvalidated data as physics-grade.

## Claims To Emphasize

- Path-level extraction can recover author-generated plotted coordinates.
- Tick-geometry calibration is often more precise than label-based manual
  calibration.
- Anchor validation is essential.
- The method is complementary to better data publication norms.
